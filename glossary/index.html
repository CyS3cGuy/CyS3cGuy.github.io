Glossary

Feeling lost with the jargons? Glossary is here to help!

Directory Link to heading

Directory is a IT-term for “Folder”. If I say “Navigate to DirA directory”, in GUI-based term, it means that I’m asking you to double click on the folder named “DirA” and open it.

Escape Character Link to heading

Escaping a character means adding a special prefix (often a backslash ) before a character to indicate that it should be treated differently than it normally would be in a given context, such as in a string, a regular expression, in URL, or in HTML.

For example, in URL, “%” is treated as a special character, because if the URL has “%”, it will do an operation called URL encoding. Examples of URL encoding:

  • %21 is exclaimation mark (!)
  • %22 is double-quote (")
  • %24 is dollar sign ($)
  • %25 is percent sign (%)

But what if we really need %? We can escape the % by introducing the “%” URL encoding, which is “%25”. So for example, if our URL somehow needs to be http://example.com/%kipling, then we can escape the “%” by making the URL http://example.com/%25kipling. So URL will parse “%25” as “%” and revert back to original URL.

Printable/Unprintable Characters Link to heading

Printable characters are characters that a human can interpret. The fact that you are able to read this glossary page and all other pages is because the words or characters/alphabets make sense to you.

Unprintable characters, on the other hand, are characters which a human won’t be able to interpret. There are actually a lot of unprintable characters, some of which we may already came across yet we are ignorant. For example, " " (whitespace), " " (tab), are unprintable characters. If you have an executable file (i.e. .exe file), try open it in Notepad and you will see a lot of junk characters (represented by some weird symbols or simply “?”) which do not make sense to a human.

The reason unprintable characters exist is because computers read characters very differently compared to human. Computers read data in binary 0 and 1 only. For example, a human can easily recognise ‘A’, but computers interpret ‘A’ as a binary combination of ‘01000001’. Of course each printable character has its own unique binary combination, but there are only a finite number of printable characters (around 150k). But with binary combination, these numbers are capable of going up even higher, reaching millions. These additional binary combinations not used by printable characters will most likely be assigned to a variety of unprintable characters.

Unprintable characters are of paramount importance in computing. It has now become a foundation of cryptography (the art of encryption/decryption). Unprintable characters are used for different purpose, for example, when we press ‘Enter’ to go to new line, we are introducing an unprintable character called “Carriage Return”, normally represented by ‘\r’ (binary combination ‘00001101’). In networking, some unprintable characters are used to signify the start or end of actual transmission.

RegEx Link to heading

Regular Expressions (RegEx) are patterns used to match and manipulate text. They allow you to search for specific sequences of characters, validate input, and extract data. Some basic components include:

  • Literal Characters: Match exact characters (e.g., a matches ‘a’).

  • Metacharacters

    Character Description
    . Any character (except newline)
    \d Any digit
    \w Any word character (letters, digits, underscore)
    ^ Start of a string
    $ End of a string

  • Quantifiers:

    Quantifier Description
    * 0 or more times
    + 1 or more times
    {n} exactly n times
    ? exactly 1 time

Some example usages:

String RegEx Matches what?
my cat is eating cat cat
phone: 1234 \d+ 1234
contact us at info@example.com \w+@\w+.\w+ info@example.com
Today is 08/13/2024 \d{2}/\d{2}/\d{4} 08/13/2024

Shell Link to heading

When you think of “shell”, think of Terminal or Command Prompt. Actually, those are interrelated. But the difference being

  • “Terminal / Command Prompt” refers to the desktop application itself
  • “Shell” refers to the interface that user can interact with. Itnalso interprets the commands that user type in into something that a computer would understand.

Shell can come in wide variety. Different shell may require different commands to achieve the same result. It also allows different kind of customisation. While the commands used may be different, they all share one similarities. It allows users to interact directly with the OS using command line feature.

Some good examples of shells across different OS platforms:

  • In Windows, we have the normal Windows Shell, or the more advanced Windows PowerShell. If you want to list out the files in Windows Shell, you would use the command dir.
  • In Unix, we have the Bourne Shell “sh” (the most primitive), the GNU Bourne-Again Shell “bash” (the most popular), C Shell “csh”, Korn Shell “ksh”, Z Shell “zsh”, etc. If you want to list out the files in bash, you would use the command ls.

SSH Link to heading

SSH stands for Secure Shell. Think of a physical computer which is online but you do not have access to the computer physically (We call that a remote computer). SSH provides a secure way to connect to that remote computer via a Terminal / Command prompt and interacting with the shell interface rather than the usual GUI-based interface.

The default port number for SSH is 22. However, custom port numbers can be setup to facilitate SSH connection.

Terminal Link to heading

Think of Terminal / Command Prompt as the black interface that allow users to input command and interact with the Operating System. Using Terminal / Command Prompt allows the users to interact with computer via shell interface. Most users are not accustomed in using the shell interface provided via Terminal / Command Prompt due to our daily lives are so deeply intertwined with the GUI-based Windows OS, i.e. mouse click, double mouse click, up/down/left/right arrow key, drag & drop, moving mouse cursor. However, in the IT world, Terminal / Command Prompt provides a more powerful feature for interaction compared to GUI-based interface (provided one has good to expert familiarity with it).

Webshell Link to heading

It is a shell, but you access the shell via a website instead of typical Terminal / Command Prompt desktop application. An example of webshell would be Azure Cloud Shell.
