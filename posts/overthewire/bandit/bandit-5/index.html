Step 1 Link to heading

bash
bandit4@bandit:~$ ls
bandit4@bandit:~$ cd inhere
bandit4@bandit:~$ ls -a

Basically, we need to know what directory we need to navigate into. For that, we need to use ls command to know the name of the directory. We also need to know what files are present within a directory.

We already know a lot about these commands from previous levels, so there is no point to repeat those. If you are unfamiliar, remember to refer back to previous levels.

After the ls -a command in “inhere”, we notice there are 10 files, excluding the "." and "..", and all the files start with hyphen "-":

bash
bandit4@bandit:~/inhere$ ls -a
.  ..  -file00  -file01  -file02  -file03  -file04  -file05  -file06  -file07  -file08  -file09

Step 2 Link to heading

bash
bandit4@bandit:~$ file ./-*

Based on the hints given in Bandit Level 4 -> 5 OverTheWire page, the password is stored in the only human-readable file, i.e. printable characters.

There are a number of ways to obtain the password. But the general idea and steps are:

  1. We first need to identify which file contains only human-readable / printable characters.
  2. Then we display the file

To achieve (1), we can use the file command. file command tells us what type of file and what kind of data it contains. Let’s take 1 file for example:

bash
bandit4@bandit:~$ file ./-file00
./-file00: data

Remember that the filename starts with hyphen so we need to append "./" to escape the hyphen. The output of the above command shows that "./-file00" has a filetype named as “data”. Let us display the file using cat ./-file00 command.

For tidiness reason, I will not show the output here, but you get the idea. It is showing a lot of gibberish, non printable characters. So if file command tells you that a file has the filetype named “data”, you know it has non-printable characters.

We continue to use file command on every single file in this directory and soon it may become a bit tedious to manually input file command and change the filename for every file. We have a better way of doing this.

bash
bandit4@bandit:~$ file ./-*
./-file00: data
./-file01: data
./-file02: data
./-file03: data
./-file04: data
./-file05: data
./-file06: data
./-file07: ASCII text
./-file08: data
./-file09: data

The command outputs all of files in the folder and its corresponding filetype. We discovered that "./file07" has filetype “ASCII”. You can think of ASCII means printable characters, contains alphabets and numbers, something that makes sense to humans. So we got our answers and we can simply cat the file to display the content. Full stop.

But the question is… what did we do to display all filetypes at one go?

Regular Expression (RegEx)

We used a technique called regular expression (a.k.a. “RegEx”) to effectively handle all the files in one go.

Imagine you have a lot of files and directories to work with, if you were to handle each file and directory one by one, you are going to waste a lot of time and effort. So most of the time, your common sense will kick in, where you try to name your files or folders according to a certain naming patterns, so that you can search using a particular keyword and all of the relevant files will appear at one go.

RegEx exploits some kind of patterns to effectively perform bulk operation on files and folders. So for example, in this case, we see that all 10 files have the pattern that always start with "./-file0" and the last character is a digit.

To obtain all the files at one go, we can use the RegEx star "*" to represents any characters (be it printable or non-printable, be it one character or multiple characters). So in this case, since we use built our RegEx to be "./-*", it means that the pattern will always start with "./-" and the "*" represents any trailing characters. Take note that the following RegEx will work as well:

  • ./-f*
  • ./-fi*
  • ./-fil*
  • ./-file*
  • ./-file0*
  • ./-??????
  • ./-f?????
  • ./-fi????

and many many more, as long as you know what does the symbol in RegEx mean. Here are 2 most common symbols in RegEx:

  • "*" means any characters, be it one or multiple characters.
  • "?" means any character, but only one character. So in example above, you can see that we have six "?" when our pattern starts with "./-"
