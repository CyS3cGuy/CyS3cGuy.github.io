Step 1 Link to heading

bash
bandit3@bandit:~$ ls
Once login, we need to first find out what files is in the directory. ls command shows that only “inhere” directory exists.

Step 2 Link to heading

bash
bandit3@bandit:~$ cd inhere

cd is a Terminal command that stands for Change Directory. We can view our current working directory by using the command pwd, which shows that we are currently at the directory named "/home/bandit3". We need to get into "/home/bandit3/inhere" directory. So we can use either of the following commands to change to the desired directory:

  • cd inhere
  • cd /home/bandit3/inhere

Step 3 Link to heading

bash
bandit3@bandit:~$ ls -la

If we use the usual ls to list all the files and directories in this directory, it will return an empty result. So it seems that the files/directories are hidden from us. In Unix system, to hide a file, we add a . in front of a filename.

To show hidden files/directories, we use:

  • ls -a to list out all files and directories, including hidden ones
  • ls -la to list out all files and directories, including hidden ones, and the corresponding permissions.

Anyway, using ls -la shows the following result:

bash
bandit3@bandit:~/inhere$ ls -la
total 12
drwxr-xr-x 2 root    root    4096 Jul 17 15:57 .
drwxr-xr-x 3 root    root    4096 Jul 17 15:57 ..
-rw-r----- 1 bandit4 bandit3   33 Jul 17 15:57 ...Hiding-From-You

Before we go into each item, we need to understand how permission in Unix system works. Supposedly we have this line item -rwxr-xr-- 3 user1 user2 33 Aug 24 15:58 exampletext

  • “exampletext” is the filename of a file.
  • “user1” is the owner of the file “exampletext”
  • “user2” is the user group associated with the file “exampletext”
  • "-rwxr-xr–" is the permission. We have to view this permission string as 4 sets:
Set Example Syntax Description
1st set - Indicates the entity type. Common types are file (represented by -) and directory (represented by d)
2nd set rwx Indicates that the owner has read (r), write w and execute x permissions. It goes without saying that it is possible to control the permission by replacing any alphabets with a - to indicate no permission given for specific action.
3rd set r-x Indicates that the user group has read (r) and execute (x) permissions, but no write (w) permission, as the permission bit is replaced by -.
4th set r-- Indicates that remaining users, excluding owner and user group, only has read (r) permission, but no write (w) and execute (x) permission, as the permission bits are replaced by -.

With this knowledge, let’s look at individual line item from ls -la command:

drwxr-xr-x 2 root root 4096 Jul 17 15:57 .

"." is a directory, represented by the “d” in front of the permission string. In fact, it represents the current directory. If we use cd . command, it essentially does nothing because it just navigate back to itself. The owner “root” have full permissions, “root” user group has read and execute permission, while remaining users have read and execute permission. Since we login as “bandit3” user, we are not “root” user or part of “root” user group. All we can do is read (e.g. copy the content, display content) and execute (e.g. navigate to the directory), but cannot write (e.g. paste new content, create new files or directories).

Note:

“root” is a super user in Unix system.

drwxr-xr-x 3 root root 4096 Jul 17 15:57 ..

".." is a directory. In fact, it represents the previous directory. If we use cd .. command, it goes back to immediate parent folder. It has the same permission as "." directory (i.e. current directory).

-rw-r----- 1 bandit4 bandit3 33 Jul 17 15:57 ...Hiding-From-You

"…Hiding-From-You" is a file, represented by the "-" in front of the permission string. What’s interesting is that:

  1. It is a hidden file, because the filename starts with ".".
  2. The owner is “bandit4”, who has read and write permission.
  3. The user group “bandit3” has only read permission.
  4. Remaining users have no permission at all.

Because we login as “bandit3”, we have read permission. So we will proceed to display the content of this file.

Step 4 Link to heading

bash
bandit3@bandit:~$ cat ...Hiding-From-You

This command display the content of "…Hiding-From-You". The file happens to contain the login password for Bandit Level 4.

Tips:

Terminal / Command prompt shell usually supports tab auto-completion. So to save time, one can simply input cat ... and press Tab key to auto complete to cat ...Hiding-From-You.

The reason why 3 dots’ are needed rather than 1 dot for tab auto-completion is because 1 dot can mean either "." (current directory), ".." (previous directory) or "…Hiding-From-You".
