Step 1 Link to heading

bash
bandit1@bandit:~$ ls
Once login, we need to first find out what files is in the directory. ls command shows that only "-" file exists.

Step 2 Link to heading

bash
bandit1@bandit:~$ cat ./-

If we use cat - command, it will not display the content, because "-" is a special character in Unix system. Which means we have to properly escape the "-" character.

For the filename that starts with a dash "-", we can use either of the 2 commands to display the content:

cat ./- : The "./" indicates current directory, "./-" indicates the "-" file in current directory. In fact, it works not just on dashed file, but it works for any file. For example, if a filename is “hello.txt”, cat ./hello.txt works as well.

cat < - : In the context of Unix system, < can either mean less than, or feeding the content of a file into a command. So cat < - means I want to feed the content in - file into cat command, so that cat command can display the content out.

The "-" file happens to contain the login password for Bandit Level 2.
